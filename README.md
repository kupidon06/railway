# Railway Digital Twin — Цифровой двойник железнодорожного узла

> **ФГАОУ ВО «Российский университет транспорта» (РУТ (МИИТ))**

> Проектная работа по дисциплине «Информационные системы управления транспортной инфраструктурой»

---

## РЕФЕРАТ

**Ключевые слова:** ЦИФРОВОЙ ДВОЙНИК, ЖЕЛЕЗНОДОРОЖНЫЙ УЗЕЛ, OSM, POSTGIS, MARTIN, MAPLIBRE GL, СИМУЛЯЦИЯ, ИСКУССТВЕННЫЙ ИНТЕЛЛЕКТ, МАРШРУТИЗАЦИЯ, A\*, SSE, DJANGO, DOCKER.

Railway Digital Twin — интеллектуальная платформа для визуализации, симуляции и прогнозирования операций на железных дорогах Российской Федерации. Система объединяет геопространственные данные OpenStreetMap, цифровое моделирование движения поездов, алгоритмы маршрутизации на основе A\* и модули искусственного интеллекта для прогнозирования задержек, обнаружения аномалий и управления инцидентами в реальном времени.

Платформа предназначена для диспетчеров, аналитиков и операторов железнодорожной инфраструктуры с целью повышения эффективности управления движением, снижения задержек и автоматизации принятия решений при инцидентах.

## 1. БАЗА ДАННЫХ И ГЕОПРОСТРАНСТВЕННЫЕ ДАННЫЕ

### Модели данных

| Модель | Приложение | Описание |
|--------|-----------|----------|
| **Node** | core | Станции, узлы (JUNCTION), депо (DEPOT) — с геокоординатами |
| **Track** | core | Железнодорожные пути между узлами (LineString) |
| **Train** | core | Поезда: номер, тип, оператор, вместимость, макс. скорость |
| **Schedule** | core | Расписания: прибытие/отправление, платформа, задержка |
| **Event** | core | События: прибытие, отправление, задержка, инцидент |
| **Incident** | core | Инциденты: тип, серьёзность, статус (ACTIVE/MONITORING/RESOLVED) |
| **TrainPosition** | twin | Текущая позиция поезда (lat/lon, скорость, статус, прогресс маршрута) |
| **MLModel** | ai | Обученные ML-модели (тип, точность, статус) |
| **PredictionResult** | ai | Результаты прогнозов (перегрузка, аномалия, задержка) |
| **SimulationScenario** | simulation | Сценарии симуляции (тип, узел, параметры) |
| **SimulationRun** | simulation | Запуски симуляции (статус, KPI, результаты) |
| **Alert** | analytics | Оповещения (тип, сообщение, статус подтверждения) |

### Импорт OSM

Данные OpenStreetMap загружаются через `osm2pgsql` в таблицы PostGIS:
- `planet_osm_point` — станции, остановки, депо
- `planet_osm_line` — железнодорожные пути (rail, subway, tram, light_rail)
- `planet_osm_polygon` — здания вокзалов, территории депо

Импорт графа: 69 462 линии, 1 292 станции, 19 151 узел (Москва и область).

## 2. СИСТЕМА МАРШРУТИЗАЦИИ

### RailwayRouter (`apps/core/routing.py`)

- Построение графа NetworkX из OSM-данных PostGIS (`planet_osm_line`)
- Алгоритм A\* поиска кратчайшего пути между двумя координатами
- Интерполяция положения поезда на маршруте по проценту прогресса
- Поддержка перестроения маршрута при блокировке участка

### IncidentRouter (`apps/ai/router.py`)

- Автоматическое определение поездов, затронутых блокировкой узла или пути
- Поиск альтернативных маршрутов через обходные узлы
- План действий: REROUTE (перемаршрутизация) или DELAY (задержка при невозможности объезда)
- Анализ конфликтов при пересечении маршрутов

## 3. ДВИЖОК СИМУЛЯЦИИ

`SimulationEngine` (`apps/simulation/engine.py`) — пошаговая симуляция сценариев:

1. Определение затронутых поездов и маршрутов на основе типа сценария
2. Запуск A\* на графе RailwayRouter для поиска альтернативных путей
3. Генерация записей TrainPosition, Schedule, Event для каждого поезда
4. Расчёт KPI: количество затронутых поездов, общая задержка (мин), конфликты

### Типы сценариев

| Тип | Описание | Параметры |
|-----|----------|-----------|
| DELAY | Задержка поезда | Поезд, минуты задержки |
| CLOSURE | Закрытие пути/узла | Узел/путь, длительность |
| CONGESTION | Перегрузка узла | Узел, коэффициент загрузки |
| INCIDENT | Авария/инцидент | Узел, тип инцидента |
| MAINTENANCE | Плановое обслуживание | Узел/путь, длительность |
| CUSTOM | Пользовательский сценарий | Произвольные параметры |

## 4. ИНТЕГРАЦИЯ ИСКУССТВЕННОГО ИНТЕЛЛЕКТА

### ML-модели (3 предобученные модели)

| Модель | Метод | Входные данные | Результат |
|--------|-------|---------------|-----------|
| **Congestion Prediction** | Статистический анализ плотности | Расписания по узлам/часам | Прогноз перегрузки (низкая/средняя/высокая) |
| **Anomaly Detection** | Z-Score обнаружение выбросов | Время в пути, задержки | Бинарная классификация (норма/аномалия) |
| **Delay Prediction** | Регрессия на исторических данных | Узел, время суток, тип поезда | Прогнозируемая задержка (мин) |

### Функционал AI

- **Создание прогноза** (`POST /api/v1/ai/predictions/create-prediction/`) — выбор поезда, узла, типа проблемы и модели
- **Маршрутизация при инцидентах** (`POST /api/v1/ai/incident-routing/`) — анализ инцидента, поиск альтернативных маршрутов, план действий
- **Обучение моделей** — `python manage.py train_models` (на основе данных расписаний и событий)

### Карта (`/map/`)

Интерактивная карта на базе MapLibre GL JS с функционалом:

- **Векторные слои OSM** (через Martin/Django-прокси): пути (rail, subway, tram), станции, остановки
- **GeoJSON-слои**: позиции поездов (обновление 200 мс), инциденты, узлы-пересечения
- **Переключение слоёв**: пути, вокзалы, остановки, разъезды, поезда, инциденты, метки
- **Тёмная / светлая тема**: переключение стиля карты
- **Поиск поезда**: поиск по номеру с выпадающим списком
- **Панель поезда**: статус, скорость, прогресс маршрута, расписание, SVG-модель поезда
- **Панель станции**: координаты, количество путей, ближайшие поезда, отправка отчёта
- **Чат с машинистом**: симуляция общения (текст, голосовые, видео)
- **Отправка инцидента**: форма с типом, серьёзностью, описанием
- **SSE-поток**: позиции поездов обновляются в реальном времени через Server-Sent Events
- **Анимация**: плавное перемещение иконок поездов с интерполяцией

## 9. ИМПОРТ ДАННЫХ

### seed_data — демо-данные

```bash
python manage.py seed_data              # загрузить
python manage.py seed_data --clear      # очистить и загрузить
```

Создаёт: 8 станций (Ленинградский, Казанский, Киевский, Белорусский, Павелецкий, Курский, Московский-СПб, Нижний Новгород), пути, 12 поездов, расписания, 40 событий, 3 инцидента, позиции поездов.

```bash
python manage.py import_osm                         # импорт всей Москвы
python manage.py import_osm --clear                 # очистить и импортировать
python manage.py import_osm --bbox "37.5,55.6,37.8,55.8"  # по BBOX
```

Импортирует реальный граф из PostGIS OSM-таблиц: станции (`planet_osm_point`), пересечения (`planet_osm_line`), пути (`Track`).

### train_models — обучение ML-моделей

```bash
python manage.py train_models
python manage.py train_models --clear
```

Создаёт 3 модели: Congestion Prediction, Anomaly Detection, Delay Prediction.

## ЗАКЛЮЧЕНИЕ

В ходе выполнения проектной работы разработан цифровой двойник железнодорожного узла — интеллектуальная платформа, объединяющая геопространственные данные OpenStreetMap, алгоритмы маршрутизации A\*, движок симуляции сценариев и модули искусственного интеллекта для прогнозирования и управления инцидентами.

Платформа обеспечивает:

- Симуляцию сценариев (задержки, закрытия, перегрузки, аварии) с расчётом KPI
- Прогнозирование перегрузок и задержек с использованием ML-моделей
- Автоматическую маршрутизацию и перепланирование при инцидентах
- Многоуровневый контроль доступа (RBAC) и безопасную аутентификацию

Поставленные цели и задачи проекта выполнены. Разработанная платформа может служить основой для дальнейшего функционального расширения и практического внедрения в системы управления железнодорожной инфраструктурой.
